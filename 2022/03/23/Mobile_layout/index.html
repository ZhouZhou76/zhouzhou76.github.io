方案一：Viewport

什么是Viewport

  1. Viewport（视口/视窗/视图）是浏览器显示页面内容的屏幕区域。
  2. 在移动端，整块显示屏就相当于Viewport，但这种说法也并不正确，因为在移动端设备中，浏览器视口并不是整个屏幕。
  3. 因此在Viewport中有三种视口,分别表示为：
    visual viewport(可视视口)
    layout viewport(布局视口)
    ideal viewport(理想视口)
  4. 为了能适配大部分设备，大部分浏览器把viewport的layout viewport(布局视口)默认宽度设为980px。
  5. layout viewport(布局视口)可以使用document.documentElement.clientWidth来获取。
  6. 由于layout viewport(布局视口)的宽度是远大于浏览器宽度的，因此需要一个新的Viewport来代表浏览器的可视区域宽度，这个视口则被称为visual viewport(可视视口)。
  7. visual viewport(可视视口)可以使用window.innerWidth来获取。
  8. ideal viewport(理想视口)并没有一个固定尺寸，不同的设备有不同的ideal viewport(理想视口)，如苹果5的ideal viewport(理想视口)是320px，苹果6s的ideal viewport(理想视口)是375px。

使用Viewport适配

安装postcss-px-to-viewport并进行postcss插件配置
npm install postcss-px-to-viewport --save-dev

兼容性问题

postcss-px-to-viewport同样存在第三方组件库兼容性问题，
如在设计稿为750px时使用vant组件库会将vant组件样式缩小，
vant组件库的设计稿是按照375px来开发的，因此viewportWidth为750px时，会出现转换问题。

方案二：rem单位

rem单位是什么？

  1. 百分比布局,也叫流式布局。
    通过盒子的宽度设置成百分比来根据屏幕的宽度来进行伸缩，不受固定像素的限制，内容向两侧填充。
    原理：
     max-width 最大宽度 max-height 最大高度
 min-width 最小宽度 min-height 最小高度
  2. 等比缩放布局,也叫rem布局。
    原理，通过meta修正1px对应的物理像素数量，在根据统一的设计稿来生成html上的动态font-size。
    单位：
     em : 是一个相对单位,1em等于当前元素或父元素的font-size值。
 rem : 是一个相对单位,1rem等于根元素的font-size值。
 vw / vh : 把屏幕分为100份,1vw等于屏幕宽的1%。

方案三：媒体查询

利用媒体查询,即media queries,可以针对不同媒体类型定义不同的样式,从而实现响应式布局。
媒体类型取值 作用
all 用于所有设备
print 用于打印机和打印预览
screen 用于电脑屏幕，平板电脑，智能手机
speech 应用于屏幕阅读器等发声设备
